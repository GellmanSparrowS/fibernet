## 一、当前可视化系统架构

FiberNet 的可视化由 3 层渲染器组成：

### 第 1 层：核心渲染器 (`viz/render.py`)

主入口是 `render_graph()`，支持：
- **4 种主题 (theme)**: `dark`, `light`, `blueprint`, `publication`
- **7 种着色模式 (color_by)**: `uniform`, `orientation`, `length`, `stress`, `strain`, `fiber`, `custom`
- **线条参数**: `line_width`, `show_nodes`, `colormap`
- **输出**: matplotlib Figure 对象，可 savefig 或进一步编辑

```python
fig = fn.render_graph(g, theme="dark", color_by="orientation", colormap="coolwarm")
```

### 第 2 层：Gallery 渲染器

`render_gallery()` 生成多图拼版，用于展示多个基元：
- 自动生成 3×4 或自定义布局
- 每个子图使用统一的主题和颜色方案

### 第 3 层：3D 渲染器

`render_graph_3d()` 使用 matplotlib 3D 投影或 pyvista（如果可用）：
- 3D scatter + line 渲染
- 深度着色 (z-coordinate mapping)

## 三、为什么"感觉很花"？

### 原因 1：Orientation 着色导致颜色爆炸

`color_by="orientation"` 把每条边的角度映射到 colormap (如 coolwarm)，
这意味着结构中有 N 条边就可能有 N 种不同颜色。

以 honeycomb 5×5 为例：130 条边 → 最多 130 种颜色。
当这些颜色映射到 deep black 背景上时，高饱和度 + 高对比度 → 视觉上非常杂乱。

**数据支持**: 01_2d_gallery 有 188 种量化颜色，是所有图片中最多的。

### 原因 2：深黑背景 + 高饱和度 = 视觉疲劳

当前 11/12 张图使用深黑背景 (RGB 10,10,15)。
在黑色背景上，即使中等饱和度的线条 (0.7-0.8) 也会显得非常鲜艳。
人眼在暗背景下对亮色特别敏感（暗视觉效应），导致视觉疲劳。

**数据支持**: 平均饱和度 0.785，在深黑背景上视觉效果约等于白底上的 0.9+。

### 原因 3：密集结构 + 多条线叠加

对于 Voronoi (216+ nodes) 或 5×5 网格等密集结构，
大量线条交叉、叠加，加上每条线不同颜色，形成"彩虹效果"。

### 原因 4：多图拼版增加信息过载

01_2d_gallery (3×4 = 12 个子图) 和 10_ml_dataset (6 个样本)
同时展示多个不同结构，每个都有自己的颜色方案，
导致整体视觉信息量过大。

### 例外：3D 和蓝图主题看起来清爽

- **3D 图 (06, 07)**: 仅 22 种颜色，前景占比低 (6-8%)，结构简单
- **蓝图主题 (03)**: 饱和度最低 (0.628)，颜色数最少 (109)，
  深蓝色背景 (10,22,40) 比纯黑更柔和

## 四、改进建议

### 方案 A：降低颜色复杂度（推荐）

1. **默认使用 `color_by="fiber"` 或 `color_by="uniform"`**
   - `fiber`: 每条纤维一种颜色（而非每条边），颜色数量大幅减少
   - `uniform`: 所有边同一颜色，最简洁

2. **使用低饱和度 colormap**
   - 替换 `coolwarm` → `viridis` 或 `cividis`（感知均匀，低饱和）
   - 或者使用自定义低饱和调色板

3. **改用浅色或柔和背景**
   - `theme="publication"`: 白底 + 细线，适合论文
   - `theme="light"`: 白底 + 深色线条
   - 或新增 `theme="soft"` 使用浅灰底 (RGB 240,240,245)

### 方案 B：分层渲染策略

1. **结构概览层**: 用统一颜色画所有边，显示整体拓扑
2. **细节层**: 放大局部区域，用 orientation/stress 着色
3. **组合展示**: 概览 + 1-2 个细节放大

### 方案 C：新增 publication 模式

为学术论文专门设计的渲染风格：
- 白底 / 浅灰底
- 黑色或深灰色线条
- 仅在需要强调时用单色高亮
- 无 glow / 无散射
- 清晰的标注和比例尺

```python
# 示例：论文风格渲染
fig = fn.render_graph(
    g,
    theme="publication",
    color_by="uniform",
    line_color="#333333",
    line_width=0.8,
    show_nodes=True,
    node_size=2,
)
```

## 五、当前各主题的视觉特征

| 主题 | 背景 | 线条色 | 适用场景 | 视觉感受 |
|------|------|--------|---------|---------|
| `dark` | 深黑 (#0A0A0F) | 多色 (colormap) | 展示/演示 | 高对比、艳丽、"花" |
| `light` | 白 (#FFFFFF) | 深灰/彩色 | 日常查看 | 清晰、传统 |
| `blueprint` | 深蓝 (#0A1628) | 蓝白色 | 技术展示 | 专业、沉稳 |
| `publication` | 白 (#FFFFFF) | 黑/深灰 | 论文插图 | 简洁、专业 |

**结论**: 当前 showcase 主要使用 `dark` 主题 + `orientation` 着色，
这是视觉效果最"花"的组合。建议日常使用 `blueprint` 或 `publication` 主题，
仅在需要展示色彩丰富性时使用 `dark` + `orientation`。
